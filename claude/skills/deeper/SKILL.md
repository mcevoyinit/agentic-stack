# /deeper — One Level Under

The first answer is usually the surface. This skill forces the next layer.

## Do this

Take the current finding/claim/design and ask, in order, until one bites:

1. **Symptom vs cause.** Is this the actual root, or a thing the root produced?
   What is one level under it?
2. **The load-bearing assumption.** What has to be true for this conclusion to
   hold? Name it, then test whether it actually is.
3. **Second-order effects.** If this is right, what does it then cause that
   isn't obvious yet? Who/what does it break downstream?
4. **The framing tax.** What does the obvious way of stating this hide? Restate
   the problem from a different angle and see what appears.
5. **The strongest counter.** What is the best case that this is wrong, and
   does it survive?

Then state the deeper answer plainly, and what it changes.

## Rules

- **One real layer beats five shallow restatements.** Don't pad. Find the
  non-obvious thing and say it.
- **Verify the deeper claim too.** Going deeper is not license to speculate.
  If the deeper answer is a hypothesis, label it and say how to test it.
- **If the surface answer survives the dig, say so** — and say why it held.
  Depth that confirms is still depth. Don't invent a hidden layer that isn't
  there.
- Insight, not effort. For finishing the work itself, that's /push.
