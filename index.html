Clone https://github.com/mcevoyinit/agentic-stack, run its install.sh, then run verify-install.sh and fix anything the doctor flags. Then open docs/CUSTOMISE.md and walk me through the setup step by step, starting with the bits that need no accounts or keys.

Claude clones it, installs it, health-checks it, then tours you through the options. Nothing needs an account or a key on day one.

What this is

This is the actual Claude Code stack I drive every day, made installable. The idea is to turn Claude Code from a chat box into an operating system for agents:

  • Skills — 66 reusable capabilities the agent picks up on a trigger word: research, planning, review, writing, multi-model deliberation.
  • Rules — an always-on layer that shapes how the agents behave.
  • Hooks — scripts that fire on session events, so useful things happen without being asked.
  • MCP servers — the bridges that let agents reach real tools: email, calendar, chat, files.
  • A shared memory layer — so every agent works from the same up-to-date picture instead of starting cold each session.

None of the pieces are exotic. The leverage is in composition — a rule defines a todo protocol, a hook auto-captures action items into it at session end, a skill reads it back the next morning. Three simple pieces compose into something that feels like memory.

Or by hand

git clone https://github.com/mcevoyinit/agentic-stack
cd agentic-stack
./install.sh          # copies everything into ~/.claude, backs up anything it would touch
./verify-install.sh   # doctor: tells you what's working and what's left to configure

Nothing gets auto-filled, no credentials are created, no databases are pre-populated. docs/CUSTOMISE.md is the guided tour of the few places you plug in your own details — you can ignore 90% of it on day one.

The rules

Small always-on instruction files. Each fixes one recurring annoyance:

draftsfiles written for humans are plain .txt, so pasting into email or chat never brings markdown junk along
fable-5how to get the best out of the model: effort levels, delegation, verification habits
kb-triggerscheck the knowledge base before answering from memory
output-formatnarrow tables and short lines so terminal output is actually readable
remindersone todo file, one protocol, so “remind me to X” just works

The hooks

Scripts that fire automatically on session events — this is where the setup starts to feel like it has a memory:

check-links.pybefore Claude finishes a reply, every link it gave you gets checked; dead or wrong-content links get flagged
session-end-autocapture.shwhen a session ends, action items get pulled out into your todo file automatically
session_end_autocapture.pythe Python worker behind the hook above
sync-session-index.pyrepairs the session index when transcripts go missing from /resume (optional, unwired by default)

The three little databases

All optional, all empty, all plain SQLite — run each folder's setup.py when you want it:

canonicala tiny “source of truth” registry. Numbers that change (a deadline, a price, a figure) get a pointer to the file that holds the truth, never hardcoded into a prompt where they silently go stale.
recall-dban empty SQLite schema for indexing your own past Claude conversations, so “what did we decide about X last month?” becomes answerable.
knowledge-basean empty schema for topic dossiers (people, companies, projects) that skills like /kb query.

MCP servers it knows about

Named with placeholder credentials only — you bring your own accounts, or use community servers:

codex conversation-search github gws hermes knowledge-base nimble slack whatsapp x

The flagship: morning-brief

A daily briefing that reads your calendar, inbox and chat, works out who you actually owe a reply to, checks your todo list against ground truth (so it stops nagging you about things you already did), and renders it all as one glanceable card. claude/skills/morning-brief/INTEGRATION.md is the staged setup guide — an email + calendar connection alone gets you 80% of the value.

Is it clean?

  • Privacy: every file passed an aggressive keyword gate plus manual sweeps for names, emails, phone numbers, wallet addresses and account details. Personal skills, memory, transcripts and credentials were excluded wholesale, never copied.
  • Security: every script was independently audited twice; real issues found were fixed and each fix proven with a crafted hostile input before shipping.
  • It actually installs: tested end to end in throwaway environments, re-run for idempotency, and health-checked by the bundled doctor — a fresh install shows zero warnings.
